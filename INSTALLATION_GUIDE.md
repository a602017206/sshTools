# sshTools 安装指南

## macOS 安装说明

如果您在打开应用时遇到"文件已损坏"或"无法打开"的提示，请按照以下步骤操作：

### 方法一：使用终端命令（推荐）

1. 打开"终端"应用（在"应用程序" → "实用工具"中）

2. 将应用拖到终端，输入以下命令：
   ```bash
   xattr -cr
   ```
   然后将 `sshTools.app` 拖到终端窗口，回车

   完整命令示例：
   ```bash
   xattr -cr /Applications/sshTools.app
   ```

3. 现在可以正常打开应用了

### 方法二：通过系统设置允许

1. 尝试打开应用（会显示错误提示）

2. 打开"系统偏好设置" → "安全性与隐私" → "通用"

3. 在底部找到关于 sshTools 的提示，点击"仍要打开"

4. 再次尝试打开应用，在弹出的对话框中点击"打开"

### 方法三：禁用 Gatekeeper（不推荐）

如果上述方法都不起作用，可以临时禁用 Gatekeeper：

```bash
sudo spctl --master-disable
```

打开应用后，建议重新启用：
```bash
sudo spctl --master-enable
```

## Windows 安装说明

直接双击运行 `sshTools.exe` 即可。

如果 Windows Defender 提示：
1. 点击"更多信息"
2. 点击"仍要运行"

## Linux 安装说明

1. 赋予执行权限：
   ```bash
   chmod +x sshTools
   ```

2. 运行应用：
   ```bash
   ./sshTools
   ```

## 常见问题

### Q: 为什么 macOS 会显示"文件已损坏"？

A: 这是 macOS 的安全机制。由于该应用未通过 Apple 官方公证，系统会默认阻止运行。使用上述方法可以安全地允许运行。

### Q: 这个应用安全吗？

A: sshTools 是开源软件，所有代码都可以在 GitHub 上审查。应用不含恶意代码。

### Q: 使用 `xattr -cr` 命令安全吗？

A: 是的，这个命令只是移除文件的"隔离"标记，不会修改应用本身。

### Q: 我需要 Apple ID 或开发者账号吗？

A: 不需要，按照上述步骤即可正常使用。
